---
title: 'Role-Based Visibility'
description: 'Control which menu items are visible to specific WordPress user roles'
---

Role-based visibility allows you to show or hide menu items based on WordPress user roles, enabling you to create personalized navigation experiences for different customer types.

## Overview

Every menu item can be restricted to one or more user roles. Items with no role restrictions are visible to everyone (including guests if they somehow access the My Account page).

<Info>
  Role filtering happens server-side during menu generation, so restricted items are completely hidden from unauthorized users—not just visually hidden with CSS.
</Info>

## How It Works

### Role Filtering Logic

When the menu is generated, the plugin checks the current user's roles against each item's allowed roles:

```php
// Role filtering (class-wmab-frontend.php:33-49)
private static function get_items_with_keys() {
    $items   = WMAB_Settings::get_menu_items();
    $current = wp_get_current_user();
    $roles   = (array) $current->roles;
    $result  = [];

    foreach ( $items as $item ) {
        if ( empty( $item['enabled'] ) ) {
            continue;
        }

        // Role filtering
        if ( ! empty( $item['roles'] ) ) {
            if ( ! array_intersect( $item['roles'], $roles ) ) {
                continue;  // Skip item if user doesn't have any matching roles
            }
        }

        // ... add item to menu
    }

    return $result;
}
```

<Steps>
  <Step title="Get Current User Roles">
    WordPress returns an array of role slugs for the logged-in user (e.g., `['customer']` or `['wholesale_customer', 'subscriber']`).
  </Step>
  
  <Step title="Check Item Roles">
    If the item has no role restrictions (`roles` is empty), it's shown to everyone.
  </Step>
  
  <Step title="Match Roles">
    If the item has role restrictions, PHP's `array_intersect()` checks if the user has at least one matching role.
  </Step>
  
  <Step title="Show or Hide">
    Items with matching roles are included in the menu. Items without matches are completely excluded.
  </Step>
</Steps>

## Configuring Role Visibility

<Steps>
  <Step title="Open Item Editor">
    Click the edit button (✏️) on any menu item, or create a new item.
  </Step>
  
  <Step title="Find Roles Section">
    Scroll to the **Visible to Roles** section in the modal.
  </Step>
  
  <Step title="Select Roles">
    Check the boxes for roles that should see this item. Leave all unchecked to show to everyone.
  </Step>
  
  <Step title="Save">
    Click **Save Item**, then **Save Menu**.
  </Step>
</Steps>

### Role Checkboxes

The builder displays all available WordPress roles as checkboxes:

```php
// Get all WordPress roles (class-wmab-settings.php:130-136)
public static function get_all_roles() {
    $roles = [];
    foreach ( wp_roles()->role_names as $slug => $name ) {
        $roles[] = [ 'slug' => $slug, 'name' => $name ];
    }
    return $roles;
}
```

This includes:
- Default WordPress roles (Administrator, Editor, Author, Contributor, Subscriber)
- WooCommerce roles (Customer, Shop Manager)
- Custom roles added by plugins (Wholesale Customer, VIP Member, etc.)

<Note>
  The checkbox grid is dynamically populated based on roles registered in your WordPress installation.
</Note>

## Multiple Role Support

An item can be visible to multiple roles simultaneously. If a user has **any** of the specified roles, they'll see the item.

**Example:**

```php
// Item configuration
[
    'title' => 'Support Portal',
    'roles' => ['premium_member', 'vip_member', 'administrator'],
]
```

Users with `premium_member` **OR** `vip_member` **OR** `administrator` can see this item.

### Role Logic: OR vs AND

The plugin uses **OR logic** for multiple roles:

```php
// Match if user has ANY of the specified roles
if ( ! array_intersect( $item['roles'], $roles ) ) {
    continue;  // No match
}
```

There's no built-in AND logic (requiring all roles). If you need that:

```php
// Custom filter for AND logic
add_filter( 'wmab_menu_items', function( $items ) {
    foreach ( $items as &$item ) {
        if ( $item['id'] === 'special-item' ) {
            // Require BOTH premium_member AND verified_user
            $user = wp_get_current_user();
            if ( ! in_array( 'premium_member', $user->roles ) || ! in_array( 'verified_user', $user->roles ) ) {
                $item['enabled'] = false;
            }
        }
    }
    return $items;
});
```

## Custom Roles

The plugin works seamlessly with custom roles added by plugins or code.

### Popular Role Manager Plugins

- **Members** — Create and manage custom roles
- **User Role Editor** — Edit role capabilities
- **WooCommerce Wholesale Prices** — Adds wholesale customer roles
- **Paid Memberships Pro** — Membership level roles
- **WooCommerce Memberships** — Membership plan roles

### Creating Custom Roles Programmatically

```php
// Add custom role on plugin activation
register_activation_hook( __FILE__, function() {
    add_role(
        'vip_member',
        __( 'VIP Member' ),
        [
            'read'         => true,
            'edit_posts'   => false,
            'delete_posts' => false,
        ]
    );
});
```

Once the role is registered, it automatically appears in the My Account Menu Builder role selector.

## Role Assignment

How users get assigned roles is outside the scope of this plugin, but common methods include:

### Manual Assignment

**Via WordPress Admin:**
1. Go to **Users → All Users**
2. Edit a user
3. Change **Role** dropdown
4. Click **Update User**

### Automatic Assignment

**On Purchase (WooCommerce):**

```php
// Assign role when specific product is purchased
add_action( 'woocommerce_order_status_completed', function( $order_id ) {
    $order = wc_get_order( $order_id );
    foreach ( $order->get_items() as $item ) {
        $product_id = $item->get_product_id();
        if ( $product_id == 123 ) {  // Replace with your product ID
            $user_id = $order->get_user_id();
            $user = new WP_User( $user_id );
            $user->add_role( 'premium_member' );
        }
    }
});
```

**On Registration:**

```php
// Assign custom role instead of default 'customer'
add_action( 'user_register', function( $user_id ) {
    $user = new WP_User( $user_id );
    $user->remove_role( 'customer' );
    $user->add_role( 'basic_member' );
});
```

## Security Considerations

<Warning>
  **Server-Side Filtering:** Role visibility is enforced server-side. Items are excluded from the menu array before rendering, so users cannot bypass restrictions with browser dev tools.
</Warning>

<Info>
  **Endpoint Access:** Even if a menu item is hidden, users can still access the endpoint URL directly if they know it. To fully restrict access, add capability checks to your endpoint content:
</Info>

```php
// Restrict endpoint access by role
add_action( 'woocommerce_account_vip-lounge_endpoint', function() {
    $user = wp_get_current_user();
    if ( ! in_array( 'vip_member', $user->roles ) ) {
        wp_die( __( 'You do not have permission to access this page.' ) );
    }
    
    // Render VIP content
    echo '<h2>Welcome to the VIP Lounge</h2>';
});
```

## Troubleshooting

### Item Not Showing for Expected Role

**Check:**
1. The user actually has the role assigned (verify in **Users → Edit User**)
2. The role slug matches exactly (case-sensitive)
3. The item is **enabled** in the builder
4. Menu changes have been saved

**Debug:**

```php
// Check current user roles
add_action( 'woocommerce_account_dashboard', function() {
    $user = wp_get_current_user();
    echo '<pre>'; print_r( $user->roles ); echo '</pre>';
});
```

### Item Showing for Wrong Role

**Possible causes:**
1. No roles selected (item visible to all)
2. User has multiple roles, one of which matches
3. Custom role has same slug as restricted role

### Custom Role Not Appearing in Builder

**Solution:** The role might not be registered yet. Ensure custom roles are added before the My Account Menu Builder settings page loads:

```php
// Register custom role early
add_action( 'init', function() {
    if ( ! get_role( 'vip_member' ) ) {
        add_role( 'vip_member', 'VIP Member', ['read' => true] );
    }
}, 1 );
```

## Advanced: Conditional Role Assignment

Combine role-based visibility with dynamic role assignment for powerful workflows:

```php
// Grant VIP role after 10 purchases
add_action( 'woocommerce_order_status_completed', function( $order_id ) {
    $order = wc_get_order( $order_id );
    $user_id = $order->get_user_id();
    
    if ( $user_id ) {
        $order_count = wc_get_customer_order_count( $user_id );
        if ( $order_count >= 10 ) {
            $user = new WP_User( $user_id );
            $user->add_role( 'vip_member' );
        }
    }
});
```

Now your "VIP Lounge" menu item (restricted to `vip_member` role) automatically appears after the 10th purchase.

## Best Practices

<Tip>
  **Default to Visible:** Only use role restrictions when necessary. Most items should be visible to all customers.
</Tip>

<Tip>
  **Clear Role Names:** Use descriptive role slugs like `wholesale_customer` instead of `wc1` for maintainability.
</Tip>

<Tip>
  **Document Roles:** Keep a list of custom roles and their purposes for future reference.
</Tip>

<Warning>
  **Don't Rely Solely on Menu Visibility:** Always add capability checks to sensitive endpoint content to prevent direct URL access.
</Warning>
