External links allow you to add menu items that point to URLs outside the My Account section—such as external websites, blog posts, support pages, or other areas of your site.

## Overview

Unlike endpoints that create new pages under `/my-account/`, link-type menu items direct users to completely different URLs when clicked.

<Info>
  Links can open in the same tab or a new tab, making them perfect for external resources, documentation, or cross-site navigation.
</Info>

## Creating External Links

<Steps>
  <Step title="Open Menu Builder">
    Navigate to **WooCommerce → Settings → My Account Menu**.
  </Step>
  
  <Step title="Add New Item">
    Click the **Add Item** button in the toolbar.
  </Step>
  
  <Step title="Select Link Type">
    In the modal, choose **Link** as the item type (instead of Endpoint).
  </Step>
  
  <Step title="Configure Link">
    - **Title:** Display name (e.g., "Help Center")
    - **URL:** Full URL including `https://` (e.g., `https://example.com/help`)
    - **Open In:** Same Tab or New Tab
  </Step>
  
  <Step title="Save">
    Click **Save Item**, then **Save Menu** to apply changes.
  </Step>
</Steps>

## Link Configuration

### URL Field

The URL field accepts any valid web address:

```php
// URL sanitization (class-wmab-settings.php:517)
'url' => esc_url_raw( $item['url'] ?? '' )
```

**Supported URL types:**

<Warning>
  Always include the protocol (`https://` or `http://`). URLs without protocols may not work correctly.
</Warning>

### Target Window

Control whether links open in the same tab or a new tab:

```php
// Target validation (class-wmab-settings.php:521)
'target' => in_array( $item['target'] ?? '', [ '_self', '_blank' ], true ) ? $item['target'] : '_self'
```

<Tabs>
  <Tab title="Same Tab (_self)">
    **Best for:**
    - Internal pages on your site
    - Pages within the same domain
    - Seamless navigation flow
    
    User navigates away from My Account page.
  </Tab>
  
  <Tab title="New Tab (_blank)">
    **Best for:**
    - External websites
    - Documentation or help resources  
    - Forms or tools you want users to keep open
    
    User stays on My Account in the original tab.
  </Tab>
</Tabs>

<Tip>
  For external links, **New Tab** is recommended to prevent users from losing their My Account session.
</Tip>

## How Links Work

### Frontend Rendering

Link items are rendered with a unique key prefix to avoid conflicts with real endpoints:

```php
// Link key generation (class-wmab-frontend.php:56-58)
case 'link':
    $key = 'wmab-link--' . $item['id'];
    break;
```

In the WooCommerce navigation array, link items use the pattern `wmab-link--{id}`.

### URL Override Filter

The plugin hooks into WooCommerce's endpoint URL filter to redirect link items:

```php
// URL override for link items (class-wmab-frontend.php:95-113)
public function filter_endpoint_url( $url, $endpoint, $value, $permalink ) {
    // Link items
    if ( 0 === strpos( $endpoint, 'wmab-link--' ) ) {
        $id    = str_replace( 'wmab-link--', '', $endpoint );
        $items = WMAB_Settings::get_menu_items();
        foreach ( $items as $item ) {
            if ( $item['id'] === $id && 'link' === $item['type'] && ! empty( $item['url'] ) ) {
                return esc_url( $item['url'] );
            }
        }
    }
    return $url;
}
```

When WooCommerce generates the menu, it calls `woocommerce_get_endpoint_url()` for each item. This filter intercepts link-type items and returns their custom URL instead.

### JavaScript Target Handling

The target attribute (_blank) is applied via JavaScript on the frontend:

```javascript
// Frontend script applies target attribute (assets/js/frontend.js)
$.each(wmabFront.items, function(key, config) {
    var $link = $('.woocommerce-MyAccount-navigation-link--' + key + ' a');
    
    if (config.target === '_blank') {
        $link.attr('target', '_blank').attr('rel', 'noopener noreferrer');
    }
});
```

<Note>
  The `rel="noopener noreferrer"` attribute is automatically added to `_blank` links for security.
</Note>

## Visual Indicators

Link items display a **"Link"** badge in the menu builder to distinguish them from endpoints:

```javascript
// Link badge rendering (admin.js:50-52)
var typeBadge = item.type === 'link'
    ? '<span class="wmab-item-badge wmab-badge-link">Link</span>'
    : '<span class="wmab-item-badge wmab-badge-endpoint">Endpoint</span>';
```

In the item metadata, external URLs are shown with an external link icon:

```javascript
// URL preview in builder (admin.js:58-60)
else if (item.type === 'link' && item.url) {
```

## Best Practices

<Tip>
  **Use HTTPS:** Always use secure URLs (`https://`) for external links to avoid browser security warnings.
</Tip>

<Tip>
  **Open External Links in New Tab:** Set target to `_blank` for external sites to prevent users from losing their My Account session.
</Tip>

<Tip>
  **Descriptive Titles:** Use clear, action-oriented titles like "View Help Center" or "Visit Blog" instead of generic "Click Here".
</Tip>

<Warning>
  **Avoid Too Many External Links:** Excessive external links can distract users from their account tasks. Use them sparingly.
</Warning>

## Troubleshooting

### Link Not Appearing in Menu

**Check:**
1. Item is **enabled** in the builder
2. No role restrictions excluding your user role
3. Menu has been saved after adding the link

### Link Opens in Wrong Tab

**Solution:** Edit the item and verify the **Open In** setting (Same Tab vs New Tab).

### URL Not Working

**Possible issues:**
1. Missing protocol (`https://`)
2. Typo in URL
3. External site is down or blocking referrers

**Test:** Copy the URL and paste it directly into your browser to verify it works.

### Link Showing as Active When It Shouldn't

**Cause:** Link URLs that match the current page URL will be styled as active by some themes.

**Solution:** Ensure your link URLs are distinct from My Account page URLs.

## Code Examples

### Programmatic Link Creation

```php
// Add external link via code
add_filter( 'wmab_menu_items', function( $items ) {
    $items[] = [
        'id'       => 'custom-help-link',
        'title'    => 'Help Center',
        'type'     => 'link',
        'url'      => 'https://help.example.com',
        'icon'     => 'dashicons dashicons-sos',
        'target'   => '_blank',
        'enabled'  => true,
        'roles'    => [],  // Visible to all
        'badge'    => '24/7',
    ];
    return $items;
});
```

### Conditional Link URL

```php
// Change link URL based on user role
add_filter( 'woocommerce_get_endpoint_url', function( $url, $endpoint ) {
    if ( $endpoint === 'wmab-link--help' ) {
        if ( current_user_can( 'wholesale_customer' ) ) {
            return 'https://wholesale.example.com/help';
        }
        return 'https://help.example.com';
    }
    return $url;
}, 10, 2 );
```

### Track Link Clicks

```javascript
// Google Analytics event tracking for external links
jQuery(document).ready(function($) {
    $('.woocommerce-MyAccount-navigation a[target="_blank"]').on('click', function() {
        var linkText = $(this).text().trim();
        gtag('event', 'click', {
            'event_category': 'My Account',
            'event_label': linkText,
            'value': 1
        });
    });
});
```

## Security Considerations

<Warning>
  **URL Validation:** The plugin sanitizes URLs using `esc_url_raw()`, but always verify external links point to trusted destinations.
</Warning>

<Info>
  **Noopener/Noreferrer:** Links opening in new tabs automatically include `rel="noopener noreferrer"` to prevent reverse tabnabbing attacks.
</Info>
